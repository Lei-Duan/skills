# Human Nature: Three Tensions × Five Levers

A decision and communication framework. Core premise: technical skills depreciate; understanding what makes humans pay attention and act does not. Whoever holds the money, opportunity, or resource you want is a human. Every GTM artifact and every opportunity evaluation should therefore be checked against human nature first, features second.

This skill has two modes. Identify which one the task calls for before proceeding:

- **Mode A — Opportunity Evaluation**: "Is this worth doing? How good is this opportunity?"
- **Mode B — Approach Design**: "How should I market / write / position / launch this?"

Both modes rest on the same base model below.

## Base model

### The Three Tensions (what captures human attention)

| Tension | What it is | Signals it's in play |
|---|---|---|
| **Survival** | Alertness to threats and problems — not just physical: financial loss, social embarrassment, missed opportunity, wasted effort | The audience has a pain they'd pay to remove; fear of falling behind |
| **Identity** | The drive to belong to a tribe, hold status, and be *seen* — humans replicate information in consciousness, not just genes | The audience wants recognition, membership, a label ("I'm a builder", "I'm a 4.0 player") |
| **Progress** | Once safe and belonging, humans crave clarity, direction, purpose, transformation | The audience asks "what's next", seeks self-improvement, buys roadmaps |

Rules of thumb:
- Pressing any one tension captures attention almost involuntarily; pressing all three creates a grip logic alone can't break.
- Most audiences are NOT ready for high-level Progress content. Meet them at the lowest unresolved tension first (Survival → Identity → Progress), then walk them up. Diagnose the audience's level before writing anything.

### The Five Levers (how to convert attention into action)

1. **Name the threat before the solution.** Problem first, always. Use the five levels of awareness to calibrate: (1) unaware of the problem → (2) problem-aware, solution-unaware → (3) solution-aware, comparing options → (4) product-aware, needs proof → (5) ready, needs a final nudge. Write to the level your audience is actually at.
2. **Mirror the identity.** "If you're [specific person with specific situation]..." — the reader must recognize themselves within the first lines.
3. **Explicitly exclude.** State plainly who this is NOT for. Exclusion manufactures belonging, forces a side, and pre-filters people who would never convert anyway.
4. **Paint the transformation.** Imagined experience activates much of the same neural circuitry as lived experience. Describe the after-state concretely enough that the reader pre-lives it. (Note: "same circuitry" is a simplification — partial overlap — don't cite it as hard science.)
5. **Give a tiny, obvious first step.** Small enough to be nearly impossible to refuse. Open loops (Zeigarnik effect): a started task creates tension that demands completion.

### The reality-test constraint (non-negotiable)

Knowledge of this framework counts for nothing until tested against a real audience. Every application of this skill must end with a concrete, dated, shippable artifact or experiment — a post, a landing page, a DM, a pricing test. If the output of using this skill is only analysis, the skill has failed. Ship, then read the response data, then iterate.

## Mode A — Opportunity Evaluation

Use when judging whether a product idea, sector, feature, channel, or career move is worth pursuing. Run this checklist and report findings explicitly:

1. **Tension mapping.** Which of the three tensions does this opportunity serve, and how directly? Opportunities serving zero tensions are utility plays — margin-compressed and automatable. Opportunities serving Identity or all three tend to command loyalty and pricing power. State which tension(s), with evidence.
2. **Audience awareness level.** At which of the five awareness levels is the target audience today? Level 1–2 audiences require expensive education (long GTM); level 3–5 audiences convert fast but face more competition. This directly predicts feedback-loop speed.
3. **Distribution-moat check.** Audience trust and distribution channels are compounding assets that do NOT erode when the underlying tech/model layer turns over — they are non-text-migratable. Ask: does pursuing this opportunity build an audience asset that outlives the product itself? An opportunity that builds distribution while it runs is worth more than its standalone P&L suggests.
4. **Wedge via unmet identity.** Competitors existing proves demand. The wedge question is: which specific identity does no incumbent make feel *seen*? If you can name that person precisely (Lever 2 material), the wedge is real; if you can only name a feature gap, it's weak.
5. **Verdict format.** Conclude with: tension(s) served / awareness level / distribution asset yes-no / wedge statement in one sentence / the single cheapest reality test and a date to run it.

Anti-pattern guard: if the evaluation starts generating a list of "competitors have full ecosystems / it's too hard" reasons, flag it explicitly — that pattern is usually fear seeking legitimacy, not analysis. The correct response is to shrink scope until failure is impossible and reframe competition as demand validation, not to abandon.

## Mode B — Approach Design (GTM / copy / positioning / narrative)

Use when producing or reviewing any persuasive artifact. Work through the levers **in order** — the sequence is the funnel:

1. Open by naming the threat, written to the audience's diagnosed awareness level (never open with the product).
2. Mirror the identity in the first 1–3 sentences ("If you're...").
3. Include one explicit exclusion ("this isn't for...").
4. Paint one concrete transformation scene — specific, sensory, time-bound. Not "improve your game" but "you walk onto the court remembering exactly what broke your backhand last Tuesday."
5. Close with one tiny first step (single link, single action, under 2 minutes).

Review checklist for existing copy: score each lever 0–2 (absent / present / sharp), identify the weakest lever, rewrite only that. For narratives (pitches, interviews): structure as threat → what I did → transformation, same levers, story form.

Tone constraint: the ethical version of this framework changes lives, it doesn't manipulate. Threats named must be real, transformations painted must be deliverable. If a lever requires exaggerating, cut the lever, not the truth.

## Integration with the 5-filter opportunity framework (if present in context)

When the user's five-filter sector framework (category newness / away from rails / compounding moat / fast feedback / competitors-but-unserved-wedge) is in play, this skill complements it — do not replace it:

- Mode A step 3 (distribution moat) extends filter #3: audience trust is a valid compounding-moat type alongside network economies, proprietary non-text data, and workflow lock-in.
- Mode A step 4 (identity wedge) is the operational method for testing filter #5.
- Mode A step 2 (awareness level) is a leading indicator for filter #4 (fast feedback).

Run both frameworks; where they disagree, surface the disagreement rather than averaging it away.
